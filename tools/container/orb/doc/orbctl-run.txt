Run a command on Linux.

If no arguments are provided, an interactive shell is started.
The default machine and/or user are used if not specified.

You can also prefix commands with "orb" to run them on Linux. For example:
    orb uname -a
will run "uname -a" on Linux, and is equivalent to: orbctl run uname -a

If you prefer SSH, use "orb ssh" for details.

To run a command on macOS from Linux, use "macctl run" instead.

To pass environment variables, set ORBENV to a colon-separated list of variables:
	ORBENV=EDITOR:VISUAL orb git commit

Paths are translated automatically when safe.
To be explicit, prefix Linux paths with /mnt/linux and macOS paths with /mnt/mac.

Usage:
  orbctl run [flags] [COMMAND] [ARGS]...

Aliases:
  run, exec, shell

Examples:
  orbctl run ls

Flags:
  -h, --help             help for run
  -m, --machine string   Use a specific machine
  -p, --path             Translate absolute macOS paths to Linux paths
  -s, --shell            Use the login shell instead of running command directly
  -u, --user string      Run as a specific user
  -w, --workdir string   Set the working directory
